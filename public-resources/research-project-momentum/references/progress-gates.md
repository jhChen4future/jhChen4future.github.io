# Progress Gates

Use these gates to prevent a research stage from producing only volume.

## Gate A — State delta
A stage passes when it changes one or more project-state elements:
- claim status;
- conceptual boundary;
- evidence weight;
- method or design;
- code or data artifact;
- decision;
- confidence;
- next-stage priority.

## Gate B — Assumption exposure
At least one inherited assumption must be:
- supported;
- weakened;
- rejected;
- narrowed to a boundary condition;
- converted into an explicit unresolved assumption.

## Gate C — Branch diversity
For substantial stages, at least two independent routes must be considered. They pass only if evidence from one route would not automatically imply evidence from the other.

## Gate D — Decision relevance
The result should enable or improve a decision, such as:
- retain or abandon a theory;
- choose a design;
- revise a variable;
- collect specific data;
- change a model;
- narrow a claim;
- run a decisive robustness check;
- restructure the paper.

## Gate E — Reusability
The stage should leave at least one reusable object:
- verified source ledger;
- conceptual map;
- design specification;
- analysis script;
- diagnostic output;
- decision memo;
- revised outline;
- project-state or handoff file.

## Suggested scoring
Score each dimension 0–2:

| Dimension | 0 | 1 | 2 |
|---|---|---|---|
| State delta | none | minor | material |
| Assumption test | repeated | indirectly touched | directly tested |
| Branch diversity | one path | superficial alternatives | independent routes |
| Decision relevance | no decision | narrows options | changes/enables decision |
| Reusability | narrative only | partially reusable | clear artifact/state |

A major stage should usually score at least 7/10 and must not score 0 on state delta or decision relevance.

Do not game the score. Explain low scores and redirect the project when progress is genuinely limited.
